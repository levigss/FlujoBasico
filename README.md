# Flujo basico
Práctica 1 - Flujo Básico de git y github poner una descripción del flujo básico


1.¿Qué es una Base de Datos Relacional?
Es una forma de organizar información en tablas que se pueden conectar entre sí. Imagina que es como un conjunto de hojas de cálculo de Excel que "se hablan" unas con otras para no repetir datos y que todo esté bien ordenado.
2.¿Qué es una Tabla?
Es la estructura básica donde se guarda el contenido. Está formada por filas y columnas. Cada tabla se enfoca en un solo tema (por ejemplo: "Alumnos", "Materias" o "Videojuegos").
3.¿Qué es una relación?
Es el "puente" o vínculo entre dos tablas. Se logra mediante campos comunes (llaves). Por ejemplo, puedes relacionar la tabla de Clientes con la de Pedidos usando un ID de cliente, para saber quién compró qué sin tener que escribir su nombre mil veces.
4.¿Qué representa una columna?
Representa un atributo o característica específica de lo que estamos guardando. Si la tabla es de "Celulares", las columnas serían: Marca, Modelo, Precio y Color.
5.¿Qué representa un renglón?
También se le llama registro o tupla. Representa un objeto individual y único en la tabla. Siguiendo el ejemplo del celular, un renglón sería: Samsung, S24, 20000, Negro. Es toda la info de un solo elemento.
6.¿Qué es un motor de base de datos relacional?
Es el software (el "cerebro") que se encarga de gestionar todo: guardar los datos, protegerlos, buscarlos cuando se los pides y asegurar que nadie borre lo que no debe. Técnicamente se llaman RDBMS.
7.Motores más populares
Aquí están los "pesos pesados" que usa todo el mundo:
MySQL (el más usado en web).
PostgreSQL (el favorito de los que quieren algo más avanzado).
Oracle Database (para empresas gigantes).
Microsoft SQL Server (el de Windows).
8.El motor ligero sin servidor
Ese es SQLite. Es súper útil porque es un solo archivo, no necesita instalación de servidor y es el que usan casi todas las apps de tu celular para guardar tus mensajes o configuraciones.
9.¿Qué significa SQL?
Significa Structured Query Language (Lenguaje de Consulta Estructurado). Es el "idioma" universal que usamos para darle órdenes a la base de datos, como: "Oye, búscame todos los usuarios que vivan en México".
10.lementos de un sistema de información (Versión Prepa)
Para que un sistema funcione, necesitas estos 5 ingredientes:
Hardware: La compu o el servidor donde vive todo.
Software: El motor (como MySQL) y la aplicación que usamos.
Datos: La info real que queremos guardar.
Usuarios: Nosotros (los que metemos o consultamos la info).
Procedimientos: Las reglas o pasos para que no se haga un relajo con los datos.
